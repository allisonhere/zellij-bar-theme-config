# Plan: Three-column swatch layout (FG / black / BG)

**Goal:** Restructure the field selector color swatches from "FG → BG → FG" to "FG → black → BG".

**Current state:**
The swatch chips in `render_field_selector` (line 362-376) render 4 columns for each field:
- When selected: ▏ (selection marker) + FG space + BG space + FG space + label + ▕
- When unselected: panel BG space + FG space + BG space + FG space

**Proposed change:**
Remove the third swatch block. Make it: FG space → black/panel-color space → BG space.
- The black space separates the FG and BG swatches visually
- Removes the redundant FG block at the end

**Files to change:**
- `src/ui/render.rs` — `render_field_selector` method, lines ~362-376

**Step-by-step:**
1. In `render_field_selector`, for the selected branch: change from `[fg space, bg space, fg space]` to `[fg space, black space, bg space]`
2. Same for the unselected branch
3. Run `cargo test` to verify

**No tests touch the field selector directly** (tests check preview pane colors, not the menu swatches), so this is a visual-only change.

**Risks:** None — purely cosmetic change to the menu rendering.
