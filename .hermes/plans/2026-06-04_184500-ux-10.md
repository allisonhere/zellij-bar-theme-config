# Plan: What would make navigation a 10/10

## Current: 7.2/10

Two-tier tabs + field selector. Fast for experts, fuzzy search is strong, but discoverability and overview suffer. Frame BG toggle is a conceptual wart.

## The ceiling isn't the features — it's the architecture

The core tension: you're navigating a MENU to change a VISUAL thing. Every keystroke in the menu is cognitive overhead that doesn't directly correspond to what you see in the preview. A 10/10 collapses that gap.

## What a 10 looks like

### 1. The preview IS the selector

Hover or highlight a preview element directly — click a pane border to edit frame colors, click tab text to edit tab colors, click a table row to edit table colors. No menu needed.

This is the holy grail. In a mouse-driven world it's trivial. In a keyboard-first TUI it's harder but doable: tab cycles through preview zones (border → text → tab → table → etc.), each zone highlights as you land on it, Enter opens the color picker, Esc goes back.

**Impact:** eliminates the mental translation layer entirely. You see what you're editing because you're editing what you're looking at.

### 2. Persistent mini-map of all fields

A compact sidebar (like an IDE file tree) showing ALL elements organized by group, with the active one highlighted. No tab switching — everything visible at all times. 20-25 chars wide, preview takes the rest.

```
 TabBar
   Tab (Selected)     ■■■
   Tab (Unselected)   ■■■
 Panes
  ▸Pane (Selected)    ■
   Text (Selected)    ■
   Pane (Unselected)  ■
   Pane (Highlight)   ■
 Content
   Table Title        ■■
   ...
 Status
   Exit (Success)     ■
   Exit (Error)       ■
   Status Bar         ■■
```

The swatch indicator (■) shows FG/BG colors inline — instant visual scan.

**Impact:** solves discoverability completely. You see everything in one glance. Navigation is ↑↓ in a flat list again, but organized.

### 3. Command palette for instant jump

`Ctrl+P` or `/` opens a text input at the top. Type "pane sel" → Pane (Selected) is the top hit. Enter to jump directly. Works from anywhere, no group switching.

This replaces both the group tabs AND the field selector as the primary navigation method for experts. The sidebar becomes read-only orientation, the palette becomes the control surface.

**Impact:** expert speed goes from "pretty good" to "instant." Two keystrokes and you're editing any element.

### 4. Inline color editing (no modal picker)

Don't open a full-screen overlay. When you select an element, a compact color editor appears below or beside the preview — sliders, hex input, color field. The preview updates in real-time as you drag. No mode switch, no context loss.

This is the biggest architectural change but the biggest UX win. The current flow is: preview → pick element → press 'c' → modal color picker → Tab to focus → adjust → Esc → back to preview. Every step is a context switch. Inline editing makes it one continuous flow.

### 5. Frame components get a proper BG

Fix the conceptual inconsistency. Frame components SHOULD have an editable background — it should control the pane fill color in the preview. `TextUnselected.background` becomes the default/fallback, but each frame component can override it.

This removes the "dead toggle" wart entirely and makes the model internally consistent. Every element has FG + BG. No special cases.

## What's achievable without a rewrite

The command palette (#3) and persistent sidebar (#2) are the high-impact, low-effort wins. They can be added to the current architecture without restructuring anything:

- **Sidebar:** Replace the two-tier tabs with an always-expanded tree view. Same data, different rendering. ~100 lines changed in `render.rs`.
- **Command palette:** `/` already exists as fuzzy search. Make it available from everywhere (not just Preview mode), and make it the primary selection mechanism. ~50 lines in `events.rs`.

Those two alone would push the score from 7.2 to ~8.5.

## The full 10

| Element | Score bump | Effort |
|---------|-----------|--------|
| Persistent sidebar tree | +1.0 | Low |
| Global command palette | +0.5 | Low |
| Preview-as-selector | +0.8 | High |
| Inline color editing | +0.5 | Very High |
| Frame BG consistency | +0.2 | Medium |

All five: **10/10**. Sidebar + palette only: **~8.7/10**.
