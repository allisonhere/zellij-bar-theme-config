# Plan: Label swatch columns with "F" and "B"

**Goal:** Add "F" and "B" characters to the color swatch blocks in the field selector menu so foreground and background columns are immediately identifiable.

**Current state:**
Each field chip in `render_field_selector` shows 3 blank colored blocks:
- `fg` block (space with BG = foreground color)
- `bg` block (space with BG = background color)  
- `fg` block again (space with BG = foreground color)

Followed by the field label text (e.g. "Tab Sel").

**Proposed approach:**
Replace the blank spaces with contrasting "F" and "B" characters on the colored backgrounds:
- First block: `"F"` on FG-colored background, with text color set to a contrasting color (dark text on light BG, light text on dark BG)
- Second block: `"B"` on BG-colored background, same contrast logic
- Third block: drop it (redundant — already planned in the previous swatch-columns plan)

Use `contrast_text()` (already exists in the codebase at line ~1296 for the color picker field marker) to determine whether to use white or dark text atop the swatch color.

**Files to change:**
- `src/ui/render.rs` — `render_field_selector`, lines ~363-376

**Step-by-step:**
1. In `render_field_selector`, for each swatch block:
   - Compute contrast text color via `contrast_text(swatch_color)`
   - Style: `Span::styled("F", Style::new().fg(contrast_text(fg)).bg(fg))` and `Span::styled("B", Style::new().fg(contrast_text(bg)).bg(bg))`
2. Apply to both the selected and unselected branches
3. Drop the redundant third FG block (per swatch-columns plan)
4. Run `cargo test`

**Validation:** `cargo test` — no tests cover the field selector menu rendering, so this is visual verification only.

**Risks:** None. Purely cosmetic.
