# Flatten project structure: move crate to repo root

## Goal

Eliminate the nested `zellij-tab-config/` subdirectory. The Rust crate lives at the repo root so the structure is flat:

```
~/projects/zellit/
├── Cargo.toml         ← was zellij-tab-config/Cargo.toml
├── src/               ← was zellij-tab-config/src/
├── deploy.sh
├── install.sh
├── install-binary.sh
├── install-source.sh
├── README.md
├── LICENSE
├── PROJECT_STATUS.md
├── loader.png, picker.png, pic.png, screen.png
├── .github/workflows/release.yml
└── .git/
```

## Step-by-step

### 1. Copy the repo root (not just the crate)

```bash
cp -a ~/Projects/zellij/zellij-tab-config ~/projects/zellit
rm -rf ~/projects/zellit/zellij-tab-config/target  # stale build cache
```

### 2. Flatten: move crate contents to repo root

```bash
cd ~/projects/zellit
# Move everything from the nested crate up one level
mv zellij-tab-config/* .
mv zellij-tab-config/.* . 2>/dev/null || true  # hidden files if any
rmdir zellij-tab-config
```

Now `Cargo.toml` is at the repo root. `cargo build` now works with no `--manifest-path`.

### 3. Update deploy.sh

- Line 52: `[ -f "zellij-tab-config/Cargo.toml" ]` → `[ -f "Cargo.toml" ]`
- Line 120-121: remove the `zellij-tab-config/` prefix
- Line 20: title text `"zellij-bar-theme-config"` → `"zellit"`

### 4. Update install.sh

- Line 74: `$TMP/repo/zellij-tab-config/Cargo.toml` → `$TMP/repo/Cargo.toml`
- Line 76: `$TMP/repo/zellij-tab-config/target/release/` → `$TMP/repo/target/release/`
- Line 4-5: update `REPO` and `BIN` vars

### 5. Update install-source.sh

- Line 26: `$TMP/repo/zellij-tab-config/Cargo.toml` → `$TMP/repo/Cargo.toml`
- Line 29: `$TMP/repo/zellij-tab-config/target/release/` → `$TMP/repo/target/release/`
- Line 4-5: update `REPO` and `BIN` vars

### 6. Update CI workflow

`.github/workflows/release.yml`:
- Line 26: `--manifest-path zellij-tab-config/Cargo.toml` → remove flag (Cargo.toml is at root)
- Line 29: `zellij-tab-config/target/...` → `target/...`
- Line 34: release asset name if binary name changes

### 7. (Optional) Update rust-version or edition

Verify `Cargo.toml` doesn't need changes — the `[[bin]]` path is already `src/main.rs` (relative), so it works unchanged.

### 8. Rename GitHub repo + update remote

```bash
cd ~/projects/zellit
gh repo rename zellit
git remote set-url origin git@github.com:allisonhere/zellit.git
```

## Risks

- **Low.** Moving files up clears the nesting. All paths in Rust source use `include_str!("../bundled_themes/…")` which are relative to `src/` — they don't depend on project root location.
- The `[[bin]]` name in Cargo.toml stays `zellij-tab-config` unless you change it — that's independent of directory structure.
