# Copy zellij-tab-config to ~/projects/zellit

## Goal

Copy the existing project from `~/Projects/zellij/zellij-tab-config/zellij-tab-config` to `~/projects/zellit` and continue development there.

## Current context

- Source: `/home/allie/Projects/zellij/zellij-tab-config/zellij-tab-config`
- Destination: `/home/allie/projects/zellit`
- Binary name (`zellij-tab-config`) comes from Cargo.toml — not from the directory name
- There are no absolute paths in the source code — all paths are relative (`../bundled_themes/`)
- The config file path is handled by `dirs` crate (system config dir), not project-location-dependent

## What to do

```bash
# Copy the project (excluding build artifacts)
cp -a ~/Projects/zellij/zellij-tab-config/zellij-tab-config ~/projects/zellit

# Delete stale build cache
rm -rf ~/projects/zellit/target

# Lost git history — optionally re-init
cd ~/projects/zellit
git init
git add -A
git commit -m "import from zellij-tab-config"
```

## Verification

```bash
cd ~/projects/zellit
cargo test
cargo run
```

## What won't transfer

- Git history (`.git/` reference is lost — re-init or set new remote)
- Cargo build cache (`target/` — delete and rebuild)
- The `.hermes/plans/` markdown files will copy fine

## Risks

- None. Copying a Cargo project is fully portable — no absolute paths, no installation-time bindings.
